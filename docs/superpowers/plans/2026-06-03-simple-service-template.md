# Simple Service Template Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a reusable React/Vite/Nginx template in `containers`, and add a `simple-service` GitOps grouping point in `home-ops`.

**Architecture:** Each simple service is an independent image built from a copied template under `containers/apps/<service-name>/`. The template builds static assets with Vite, serves them through unprivileged Nginx on port 8080, and relies on `home-ops/k8s/apps/common/simple-service/ks.yaml` to centrally register deployed services. No backend, secret, PVC, or cluster-direct operation is introduced.

**Tech Stack:** React 19, Vite 8, TypeScript 6, Tailwind CSS v4, `nginxinc/nginx-unprivileged:1.29-alpine`, Flux Kustomization, bjw-s app-template.

- This template is for lightweight single-purpose frontend services: landing pages, query tools, converters, generators, calculators, and similar no-backend apps.

- [ ] **Step 4: Write `Dockerfile`**

Create `templates/simple-service/Dockerfile`:

```dockerfile
# syntax=docker/dockerfile:1.7

FROM node:24-alpine AS builder
WORKDIR /app

COPY package*.json ./
RUN npm ci

COPY . .
RUN npm run build

FROM nginxinc/nginx-unprivileged:1.29-alpine

COPY nginx.conf /etc/nginx/conf.d/default.conf
COPY --from=builder --chown=101:101 /app/dist /usr/share/nginx/html

EXPOSE 8080
```

Reason: `nginxinc/nginx-unprivileged` runs as non-root and listens on 8080, matching Kubernetes least-privilege defaults without adding capabilities.

- [ ] **Step 5: Write `docker-bake.hcl`**

Create `templates/simple-service/docker-bake.hcl`:

```hcl
target "docker-metadata-action" {}

variable "APP" {
  default = "simple-service-template"
}

variable "VERSION" {
  default = "0.1.0"
}

variable "SOURCE" {
  default = "https://github.com/<owner>/containers"
}

group "default" {
  targets = ["image-local"]
}

target "image" {
  inherits = ["docker-metadata-action"]
  labels = {
    "org.opencontainers.image.source" = "${SOURCE}"
  }
}

target "image-local" {
  inherits = ["image"]
  output = ["type=docker"]
  tags = ["${APP}:${VERSION}"]
}

target "image-all" {
  inherits = ["image"]
  platforms = [
    "linux/amd64",
    "linux/arm64"
  ]
}
```

When copied to `apps/<service-name>/`, change `APP` to the service directory name and `SOURCE` to the repository URL if needed.

- [ ] **Step 6: Write `nginx.conf`**

Create `templates/simple-service/nginx.conf`:

```nginx
server {
    listen 8080;
    server_name _;

    root /usr/share/nginx/html;
    index index.html;

    gzip on;
    gzip_vary on;
    gzip_min_length 256;
    gzip_types
        text/plain
        text/css
        application/json
        application/javascript
        application/xml
        image/svg+xml;

    location = /healthz {
        access_log off;
        add_header Content-Type text/plain;
        return 200 "ok\n";
    }

    location /assets/ {
        try_files $uri =404;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

## Creating a new service

From the `containers` repo root:

```bash
cp -r templates/simple-service apps/<service-name>
```

Then update:

- `apps/<service-name>/docker-bake.hcl`: set `APP` to `<service-name>`.
- `apps/<service-name>/package.json`: set `name` to `<service-name>`.
- `apps/<service-name>/index.html`: set `<title>` and meta description.
- `apps/<service-name>/src/App.tsx`: replace starter content with the service UI.

After the image exists, add the Kubernetes deployment under:

```text
k8s/apps/common/simple-service/<service-name>/
```

and register it in:

```text
k8s/apps/common/simple-service/ks.yaml
```
```
