Architecture · home-ops

home-ops 架构总览

仓库是主要配置入口。Taskfile 与 Ansible 完成节点引导，Flux 从 k8s/clusters/staging 继续分发到 k8s/infra/staging 与 k8s/apps/staging；密钥链路同时覆盖 Azure Key Vault 与 repo 内 SOPS 密文；网络由 Cilium、Multus、Envoy Gateway 与家庭网络入口组成；持久化、备份、观测与集群外 compose 服务共同支撑家庭服务。

Control plane

GitOps 主干

  • 仓库入口在 k8s/clusters/staging，Flux 继续分发到 k8s/infra/staging 与 k8s/apps/staging。
  • Taskfile 与 Ansible 负责节点初装、CRD 与基础文件，不承担长期配置事实源角色。
  • 仓库还承载 docs、compose、ansible、bootstrap，形成完整运维工作区。

Data + network

网络与状态层

  • Cilium 负责集群网络，Multus 负责 VLAN、mDNS 与 IPv6 直连场景。
  • Envoy Gateway、caddy-external、Cloudflare DNS 与 Tailscale 组成当前入口与远程接入面。
  • CloudNative-PG、Dragonfly、Longhorn、SMB 与 Volsync 负责状态服务与复制。

Edge + runtime

集群外协作面

  • OpenWrt 提供 VLAN 划分、mDNS 反射与 BGP，承接家庭网络语义。
  • compose/ 分成 compose/vps 与 compose/sakamoto，承接代理、日志、备份与对象存储等集群外职责。
  • 业务层覆盖家庭自动化、媒体、知识与通用工具，典型例子见 Home Assistant、Jellyfin、Memos、Frigate。
